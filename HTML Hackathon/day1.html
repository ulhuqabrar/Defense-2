  2. DNS Workflow

  3. IP Address

    • Provided by ISP (e.g., Jio, Vodafone).
    • Dynamic (changes) vs Permanent (MAC address)
      • IPv4: is 32-bit IPv6 : 128 bit number
      range: 0.0.0.0 – 255.255.255.255 Range → 0.0.0.0 – 255.255.255.255
      (~4.3 billion addresses) 2¹²⁸ IP addresses
      Can allocate 2³² / 4.3 billion addresses / systems. (340 – undecillion)
      Written in hexadecimal form
      Format → 8bit | 8bit | 8bit | 8bit xxxx:xxxx:xxxx:xxxx:xxxx:xxxx:xxxx:xxxx
      0000:0000:0000:0000:0000:0000:0000:0000

    • MAC Address

      Permanent, used in Medium Access Control.

      48 bit with 12 hexadecimal digits

      in four format :-

      1. Colon separated (common)
      2. Hyphen separated (windows)
      3. Period separated (cisco & others)
      4. No separation (least common)

      A device can have multiple Mac address

      Mac – Sends / receives data

    • System

      1. You type a website name (like google.com).
      2. Request Server (DNS Resolver): Your system sends the query to its configured DNS resolver (usually your ISP’s recursive server).
      3. Root Server: If the resolver doesn’t have the answer cached, it asks a Root DNS server.
      4. TLD Server: The Root server directs the resolver to the Top Level Domain (TLD) server (like .com, .in).
      5. Authoritative Server: The TLD server points to the authoritative server for the domain (e.g., google’s authoritative nameserver).
      6. Response Back: The authoritative server returns the IP address of the requested website.
      7. System: The resolver gives the IP back to your system, which can now contact the website directly



    • When that TLD server takes back to the Authoritative Server which gives DNS (IP address) of the website which fetches back & is stored in the system’s recursive server.


    • Port No.:

      Part of system that requests certain data from internet or sends.

      • http → 80
      • https → 443 (Secure, others can’t read)

      16 bit binary digit

      No. of possible ports = 0 to 65,536 (2¹⁶)

      No. of Identifiers : (0–65,536)²¹⁶ – 1

      0 – 1023 → Port No. are reserved.

      1024 – 49151 → Registered ports (reserved)

      49152 – 65535 → Dynamic


    • System checks for DNS server in itself first. If not found checks in recursive server.
      • Recursive also checks in itself first, if not found.
      • First goes to root server to find end domain (.com, .in) & gives to appropriate TLD server address.

    • LAN

      Multiple systems share data via connected switch. (Scalable traffic)

      Router connects LANs. They are connected to switches and another routers

      3. 192.168.0.0 – 192.168.255.255 (Usually small companies & local ISP)
