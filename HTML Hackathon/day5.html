Day 5 :Forms

Topics Covered:

  1. Forms



Written as :

<form>
<label></label
<input type="text/No.">
<br>
<input type="radio">
<label for="book">Book</label>
<input type="radio" id="book" name="Book" placeholder="Name">
</form>

practice - excalidraw.com - will help in interviews

For database, to know the value we are sending, we need to send both name & value from the form.

Types :
  • radio - single select
  • text - only text
  • Number - only No.
  • Password - hides text
  • submit - button

    • Value should be given in form radio; for system/backend to know the actual value we are selecting

  • Name: assigns value to given name
  • Required: Makes mandatory to fill.
  • Checkbox - can select multiple options
  • id: same as label for to group
  • Placeholder: Gives ex of what to fill when input is empty
  • Select: Gives dropdown option - present in select (many), add (value) attribute
  • Textarea: large text box to fill dialog text, can give rows, cols attribute

    • Example: <textarea name, id, rows, col>

  • Button: <button></button> used best for JavaScript
